Hive映射HBase表字段过多报错

columns has 234 elements while hbase.columns.mapping has 92 elements (counting the key if implicit))

根本原因：
对于4000个字符，hive Metastore中SERDE_PARAMS表中PARAM_VALUE字段的字符限制是此问题的根本原因。
此限制可防止Hive创建具有高列数的表，最终导致desc或select * from失败并出现上述错误。

解决方法：通过在hive Metastore中执行以下操作可以解决此问题
alter table SERDE_PARAMS MODIFY PARAM_VALUE VARCHAR(65535);
